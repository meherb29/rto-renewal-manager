# RTO Insurance Renewal Manager

An internal web application for managing vehicle insurance renewal cycles across multiple branches of an automotive dealership. Built to replace a manual Excel-based process that caused renewals to be missed when customers took their insurance elsewhere.

## Business problem

Insurance renewals for hundreds of vehicles were tracked manually in Excel across multiple branches. The system had no automatic alerts, no follow-up tracking, and no way to know when a customer had lapsed. This application centralizes renewal tracking, automates reminders, and generates insurer document packages.

## Features

- **Dashboard** — real-time view of overdue, urgent (7 days), and upcoming (60 days) renewals
- **Customer management** — supports individual and company customers, with GST/PAN tracking for companies
- **Vehicle management** — tracks vehicle specs (chassis, engine, registration, passing type) per customer
- **Renewal tracking** — full renewal lifecycle with follow-up logging (called, WhatsApp sent, quote sent, confirmed)
- **PDF generation** — generates insurer document packages with customer, vehicle, and policy details plus embedded KYC documents
- **Document upload** — stores Aadhaar, PAN, GST, and tax invoice per customer
- **Excel export** — exports all renewal data for backup or integration with other business software
- **Excel import** — one-time import of existing data from legacy Excel files
- **Role-based access** — admin sees all branches; employees see only their assigned renewals
- **Employee management** — admin can add and edit employee accounts

## Tech stack

- **Backend** — Python, Flask, SQLAlchemy, Flask-Login, Flask-Migrate
- **Database** — SQLite
- **Frontend** — Jinja2 templates, HTML, CSS
- **Other** — openpyxl (Excel import/export), Werkzeug (password hashing)

## Running locally

**1. Clone the repo**

```bash
git clone https://github.com/meherb29/rto-renewal-manager.git
cd rto-renewal-manager
```

**2. Create and activate a virtual environment**

```bash
python -m venv venv
venv\Scripts\activate        # Windows
source venv/bin/activate     # Mac/Linux
```

**3. Install dependencies**

```bash
pip install -r requirements.txt
```

**4. Create a `.env` file in the project root**

```
SECRET_KEY=your-secret-key-here
DATABASE_URL=sqlite:///rto.db
```

**5. Set up the database**

```bash
flask db upgrade
```

**6. Seed initial data**

```bash
python seed_insurers.py    # adds the insurer list
python seed_admin.py       # creates the admin account
python seed.py             # adds demo data
```

**7. Run the app**

```bash
python run.py
```

Visit `http://127.0.0.1:5000` and log in with `admin@demo.com` / `admin123`.

## Database schema

7 tables: `customer`, `vehicle`, `insurer`, `employee`, `renewal`, `followup_log`, `document`

- One customer owns many vehicles
- One vehicle has many renewal cycles
- Each renewal tracks its insurer, assigned employee, policy dates, and full follow-up history
- Documents (Aadhaar, PAN, GST, tax invoice) are stored per customer and embedded in generated PDFs
